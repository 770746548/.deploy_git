一、vue3概述

（1）为什么要学习vue3

vue目前是前端框架中比较火热的，有了vue2为什么还要学习vue3呢，就像打游戏一样，为什么要升级呢，虽然公司大部分使用的还是vue2，还是有少部分公司使用vue3，提前了解使用vue3 也不是坏处

（2）vue3的新变化

1.性能提升，首次渲染更快，diff算法更快，内存占用更少，打包体积更小，

2.更好的Typescript支持(在vue下写TS更方便了)

3.提供新的写代码方式：Composition API (需要成本学习)

（3）破坏性语法的更新

vue3.0对于vue2.0的版本大部分语法都是可以兼容使用的，但是有些破坏性语法的更新

1.移除了$on方法（eventBus现有模式不再支持，可以使用第三方插件代替）

2.移除过滤器选项(插值表达式里不能在使用过滤器filter，可以使用methods替代)

3.移除.sync语法（v-bind不能在使用.sync修饰符了，现在和v-model语法合并了）

（1）创建vue3项目

 1.vue2.0中是直接创建了一个vue实例

 2.vue3.0中按需导出了一个createApp （ceateApp做了什么）

也就是说 在vue2.0中必须要有一个根元素，在vue3中没这个要求

三、组合式API（Composition API）和选项式API（options API）

vue3 组合式api（Composition API）的出现是用来区别于vue2选项式api（options API）

下面是来区别vue2和vue3的区别代码 需求 实现盒子显示与隐藏

区别
options API（Vue2）Composition API（Vue3）
优点方便理解，逻辑清楚(数据是放在data里，方法是放在methods里)维护性强，数据、方法放在一起
缺点代码横跳，不好复用代码量大可以逻辑拆分

拆分setup  适用于代码量较多的情况  优点 分散定义，组合使用

  1. <script>
  2. function useShow() {
  3.       let show = ref(true);
  4.      const isHide = () => (show.value = false)
  5.      const isShow = () => (show.value = true)
  6.   return {show,isHide,isShow}
  7.   }
  8. import { ref } from "vue";
  9. export default {
  10.   setup() {
  11.     const {show,isHide,isShow} = useShow()
  12.     return { show, isShow, isHide };
  13.   },
  14. };
  15. </script>

四、组合式API（Composition API）

（1）setup

① 一个组件选项，在创建组件之前执行，一旦 props 被解析，并作为组合式 API 的入口点

② setup中不能使用不能使用this、this执行undefined

③ setup函数在beforecreated钩子函数前执行

④ setup函数只会在初始化的时候执行一次

  1.   beforeCreate() {
  2.     console.log("beforeCreate");
  3.   },
  4.   setup() {
  5.     console.log("setup");
  6.   },

 此时setup中的this还不是组件实例，此时this是undefined

  1. setup(props, context) {
  2.  
  3. }

① 第1个参数为props。props为一个对象，内部包含了父组件传递过来的所有prop数据
② 第2个参数为一个对象context。context对象包含了attrs，slots， emit属性，
③ 如果在data()中也定义了同名的数据，则以setup()中为准。

后面做父子双向绑定时细说参数的用法

（2）生命周期钩子

选项式 APIHook inside setup 组合式API
beforeCreateNot needed*
createdNot needed*
beforeMountonBeforeMount
mountedonMounted
beforeUpdateonBeforeUpdate
updatedonUpdated
beforeUnmountonBeforeUnmount
unmountedonUnmounted
errorCapturedonErrorCaptured
renderTrackedonRenderTracked
renderTriggeredonRenderTriggered

deactivatedonDeactivated

这些生命周期钩子注册函数只能在 setup() 期间同步使用，因为它们依赖于内部全局状态来定位当前活动实例 (此时正在调用其 setup() 的组件实例)。在没有当前活动实例的情况下调用它们将导致错误。

组件实例上下文也是在生命周期钩子的同步执行期间设置的，因此在生命周期钩子内同步创建的侦听器和计算属性也会在组件卸载时自动删除。

因为 setup 是围绕 beforeCreate 和 created 生命周期钩子运行的，所以不需要显式地定义它们。换句话说，在这些钩子中编写的任何代码都应该直接在 setup 函数中编写。

可以通过在生命周期钩子前面加上 “on” 来访问组件的生命周期钩子。前提需要引入

  1. export default {
  2.   setup() {
  3.     // mounted
  4.     onMounted(() => {
  5.       console.log('Component is mounted!')
  6.     })
  7.   }
  8. }

五、响应式ref函数

（1）不使用ref函数

  1. <template>
  2.   <div>{{ num }}</div>
  3.   <button @click="addNum">加1</button>
  4. </template>
  5.  
  6. <script>
  7. export default {
  8.   setup() {
  9.       let num=1,
  10.       addNum=()=>{
  11.           num++
  12.           console.log(num);
  13.       }
  14.       return {num,addNum}
  15.   },
  16. };
  17. </script>

点击加1后发现 视图上并没有发生改变 也就是说setup函数中返回的数据并不是响应式的

（2）使用ref函数

简单数据类型

  1. <script>
  2. import {ref} from 'vue'
  3. export default {
  4.   setup() {
  5.       let num=ref(1),
  6.       addNum=()=>{
  7.           num.value++
  8.           console.log(num);
  9.       }
  10.       return {num,addNum}
  11.   },
  12. };
  13. </script>

复杂数据类型

  1. <template>
  2.   <div>
  3.     {{ obj.a }}
  4.     <button @click="addNum">加1</button>
  5.   </div>
  6. </template>
  7.  
  8. <script>
  9. import {ref} from 'vue'
  10. export default {
  11.   setup() {
  12.       const obj = ref({a:1}) 
  13.       const addNum = ()=>obj.value.a++
  14.       return {obj,addNum}
  15.   },
  16. };
  17. </script>

使用步骤 

① 从vue框架中导入ref函数

② 使用ref函数并传入数据(简单数据类型或复杂数据类型)

③ 在setup函数中把ref函数调用完毕的返回值以对象的形式返回出去

ref 会返回一个可变的响应式对象，该对象作为一个响应式的引用维护着它内部的值，这就是 ref 名称的来源。该对象只包含一个名为 value 的 property  注意：在setup函数中需要加.value，在template模板中不需要加value

 （3）ref引用

在vue2中获取 DOM元素和 子组件的引用直接绑定ref 通过$refs.ref绑定属性名获取引用

在vue3中 直接绑定ref 是获取不到的

  1. <template>
  2.   <div ref="box"></div>
  3. </template>
  4.  
  5. <script>
  6. import { ref, onMounted } from "vue";
  7. export default {
  8.   setup() {
  9.     const box = ref(null);
  10.     onMounted(() => {
  11.       console.log(box.value);
  12.     });
  13.     return { box };
  14.   },
  15. };
  16. </script>

 注意获取dom要在onMounted钩子函数中 获取，或者添加一个点击事件

 六、reactive函数

我们使用ref函数进行定义响应式数据每次使用都要加.value，reactive就是用来省去加.value，但是只能定义复杂数据类型的数据

  1. <template>
  2.   <div>
  3.     {{ obj.a }}
  4.     <button @click="addNum">加1</button>
  5.   </div>
  6. </template>
  7.  
  8. <script>
  9. import {reactive} from 'vue'
  10. export default {
  11.   setup() {
  12.       const obj = reactive({a:1}) 
  13.       const addNum = ()=>obj.a++
  14.       return {obj,addNum}
  15.   },
  16. };
  17. </script>

七、toRefs函数 

定义完响应式数据后，我们每次都要在模板中 通过对象.的形式进行渲染数据，但是如果为了解决这种麻烦 可以通过toRefs函数进行解构，模板中就可以直接使用a了   toRefs与reactive配套使用。

  1. <template>
  2.   <div>
  3.     {{ a }}
  4.     <button @click="addNum">加1</button>
  5.   </div>
  6. </template>
  7.  
  8. <script>
  9. import {reactive,  toRefs} from 'vue'
  10. export default {
  11.   setup() {
  12.       const obj = reactive({a:1}) 
  13.       let {a} = toRefs(obj)
  14.       const addNum = ()=>obj.a++
  15.       return {a,addNum}
  16.   },
  17. };
  18. </script>

从上面看就是给a 进行解构 但是为什么不直接...obj呢 

  1.   setup() {
  2.       const obj = reactive({a:1}) 
  3.     //   let {a} = toRefs(obj)
  4.       const addNum = ()=>{obj.a++, console.log(obj.a);}
  5.       return {...obj,addNum}
  6.   },

答案是不能的  原因就是使用...a后 数据并没有保留响应式的特点

 官网：当从组合式函数返回响应式对象时，toRefs 非常有用，这样消费组件就可以在不丢失响应性的情况下对返回的对象进行分解/扩散

  如何取舍ref和reactive

 我们现在有两种定义响应式数据的方式到底该用哪个呢，

① 优先使用ref函数 因为ref 函数 可以处理简单数据类型 也可以处理复杂数据类型 ，常用于简单数据类型定义响应式数据

  ref特点： 在代码中获取或修改值时，需要补上.value 但是 template模板中不需要

② reactive 常用与定义复杂数据类型作为响应式数据

reactive  特点  在代码中获取或修改值是不需要加.value , 如果明确知道对象中有什么属性就用reactive

八、computed计算属性

（1）基本用法

语法：

  1. import { computed } from 'vue'
  2.  
  3. const 计算属性名 = computed(() => {
  4.   return 响应式数据相关计算
  5. })
  1. <template>
  2.   <div>姓名:{{name}} 月薪:{{salary}}年薪:{{total}}</div>
  3. </template>
  4.  
  5. <script>
  6. import { computed, ref } from "vue";
  7. export default {
  8.   setup() {
  9.     const name = ref("奥特曼");
  10.     let salary = ref(10000)
  11.     const total = computed(()=>salary.value * 12)
  12.     return { name,salary,total };
  13.   },
  14. };
  15. </script>

（2）高级用法

  1. <template>
  2.   <div>
  3.     姓名:{{ name }} 月薪:{{ salary }}年薪:{{ total }}
  4.     <input type="text" v-model="total" />
  5.   </div>
  6.   <button @click="double">月薪double</button>
  7. </template>
  8.  
  9. <script>
  10. import { computed, ref } from "vue";
  11. export default {
  12.   setup() {
  13.     const name = ref("奥特曼");
  14.     let salary = ref(10000);
  15.     const total = computed({
  16.         set(val){
  17.           return  salary.value=val/12
  18.         },
  19.         get(){
  20.          return salary.value * 12
  21.         }
  22.     });
  23.     const double = () => salary.value *= 2;
  24.     return { name, salary, total, double };
  25.   },
  26. };
  27. </script>

计算属性也可以绑定v-model

九、watch监听

语法：

watch(数据|数组|get函数，(新值，旧值)=>{回调处理逻辑}, {immediate:true|false, deep: true|false})

（1）单个数据监听

  1. <template>
  2.   <div>{{ num }}</div>
  3.   <button @click="addNum">加1</button>
  4. </template>
  5.  
  6. <script>
  7. import { ref, watch } from "vue";
  8. export default {
  9.   setup() {
  10.     let num = ref(1),
  11.       addNum = () => {
  12.         num.value++;
  13.       };
  14.     watch(num, (newValue) => {
  15.       console.log(newValue);
  16.     });
  17.     return { num, addNum };
  18.   },
  19. };
  20. </script>

（2）多个数据监听

  1. <template>
  2.   <div>{{ num }}||| {{num2}} </div>
  3.   <button @click="addNum">加1</button>
  4.   <button @click="addNum2">加1</button>
  5. </template>
  6.  
  7. <script>
  8. import {ref,watch} from 'vue'
  9. export default {
  10.   setup() {
  11.       let num=ref(1)
  12.       let num2=ref(2)
  13.     const addNum=()=>num.value++
  14.     const addNum2=()=>num2.value++
  15.       watch([num,num2],(newValue)=>{
  16.           console.log(newValue);
  17.       })
  18.       return {num,num2,addNum,addNum2}
  19.   },
  20. };
  21. </script>

（3）复杂数据类型监听

如果监听 复杂数据类型 在第三个参数对象中的deep:true

  1. <template>
  2.   姓名:{{ stu.name }},公司:{{ stu.company }}
  3.   月薪 {{ stu.money.salary }}
  4.   <button @click="stu.company ='CSDN'">跳槽到CSDN</button>
  5. </template>
  6.  
  7. <script>
  8. import { reactive, watch } from 'vue'
  9. export default {
  10.   setup() {
  11.     const stu = reactive({
  12.       name: '小王',
  13.       company:'JD',
  14.       money: {
  15.         salary: 1800
  16.       }
  17.     })
  18.     watch(stu, () => {
  19.       // 数据变化之后的回调函数
  20.       console.log('stu发生了变化')
  21.     })
  22.     return { stu }
  23.   }
  24. }
  25. </script>

（4）监听复杂数据类型中的属性

写watch的时候避免使用deep引起性能的问题，第一个参数可以使用回调的方式

  1. <template>
  2.   姓名:{{ stu.name }},公司:{{ stu.company }}
  3.   月薪 {{ stu.money.salary }}
  4.   <button @click="stu.money.salary =18000">月薪上升</button>
  5. </template>
  6.  
  7. <script>
  8. import { reactive, watch } from 'vue'
  9. export default {
  10.   setup() {
  11.     const stu = reactive({
  12.       name: '小王',
  13.       company:'CSDN',
  14.       money: {
  15.         salary: 1800
  16.       }
  17.     })
  18.     watch(()=>{
  19.         return stu.money.salary
  20.     }, () => {
  21.       // 数据变化之后的回调函数
  22.       console.log('薪资发生变化了')
  23.     })
  24.     return { stu }
  25.   }
  26. }
  27. </script>

（5）开启深度监听

首次监听是immediate   深度监听 deep

  1. <template>
  2.   姓名:{{ stu.name }},公司:{{ stu.company }}
  3.   月薪 {{ stu.money.salary }}
  4.   <button @click="stu.money.salary*=2">工资double</button>
  5.   <button @click="stu.money.bonus*=2">奖金double</button>
  6. </template>
  7.  
  8. <script>
  9. import { reactive, watch } from 'vue'
  10. export default {
  11.   setup() {
  12.     const stu = reactive({
  13.       name: '小王',
  14.       company:'JD',
  15.       money: {
  16.         salary: 18000,
  17.         bonus:15000
  18.       }
  19.     })
  20.     watch(() => {
  21.       return stu.money
  22.     }, () => {
  23.       console.log('stu.money发生了变化')
  24.     }, {
  25.       deep: true //  只有为true，才能监听money下所有属性的变化
  26.     })
  27.     return {
  28.       stu
  29.     }
  30.   }
  31. }
  32. </script>

十、父子通信

（1）父传子

在setup函数中有两个参数 第一个参数为props，第二个参数为context

props为一个对象，props接收后，内部包含了所有传递过来的prop数据，context包含了attrs，slots，emit属性，其中emit方法可以完成子传父

父

  1. <template>
  2.   <div>
  3.     <Son :msg="msg"></Son>
  4.   </div>
  5. </template>
  6.  
  7. <script>
  8. import { ref } from 'vue';
  9. import Son from "./son.vue";
  10. export default {
  11.   components: {
  12.     Son,
  13.   },
  14.   setup() {
  15.       const msg = ref('奥特曼')
  16.         return {msg}
  17.   },
  18. };
  19. </script>
  20.  
  21. <style scoped>
  22. </style>

子

  1. <template>
  2.   <div>子组件 {{msg}} </div>
  3. </template>
  4.  
  5. <script>
  6. export default {
  7.     props:{
  8.         msg:{
  9.             type:String,
  10.             default:''
  11.         }
  12.     },
  13.     setup(props,context){
  14.         console.log(props,context);
  15.     }
  16. };
  17. </script>
  18.  
  19. <style scoped>
  20. </style>

（2）子传父

注意vue2中的this.$emit 不在生效   setup函数中没有this ，而在setup第二个参数中有emit方法

子

  1.     setup(props,context){
  2.         context.emit('name','奥特曼')
  3.     }
  4.  
  5. //结构写法
  6. //    setup(props,{emit}){
  7. //       emit('name','奥特曼')
  8. //   }

父

  1. <template>
  2.     <Son @name="name" ></Son>
  3. </template>
  4.  
  5.   setup() {
  6.       const name = (name)=>console.log(name);
  7.         return {name}
  8.   },

  • 父传子：在setup中使用props数据 setup(props){  props就是父组件数据 }

  • 子传父：触发自定义事件的时候emit来自 setup(props,{emit}){  emit 就是触发事件函数
